## 6. Ferramentas e Boas Práticas

### 6.1 Ferramentas Recomendadas

**Biblioteca de Internacionalização:**
- Considerar migrar para `react-i18next` ou `next-intl` para gerenciar traduções de forma centralizada
- Criar arquivo `pt-BR.json` com todas as traduções
- Usar hooks `useTranslation()` nos componentes

**Exemplo de Estrutura:**
```json
{
  "auth": {
    "login": {
      "title": "Entrar",
      "email": "E-mail",
      "password": "Senha",
      "submit": "Entrar",
      "forgotPassword": "Esqueceu a senha?"
    }
  },
  "dashboard": {
    "welcome": "Bem-vindo de volta",
    "createResume": "Criar Novo Currículo"
  }
}
```

### 6.2 Boas Práticas

1. **Consistência de Termos:**
   - "Resume" sempre traduzir como "Currículo" (não "CV")
   - "Dashboard" sempre traduzir como "Painel"
   - "Settings" sempre traduzir como "Configurações"

2. **Formalidade:**
   - Usar "você" ao invés de "tu"
   - Manter tom profissional mas acessível
   - Evitar gírias ou expressões regionais

3. **Botões e CTAs:**
   - Usar verbos no infinitivo: "Criar", "Editar", "Salvar"
   - Manter textos curtos e diretos
   - Preferir "Cadastre-se" ao invés de "Registre-se"

4. **Mensagens de Erro:**
   - Ser específico sobre o problema
   - Oferecer solução quando possível
   - Exemplo: "E-mail inválido. Verifique o formato." ao invés de "Erro"

5. **Datas e Números:**
   - Formato de data: DD/MM/AAAA
   - Moeda: R$ 10,00 (vírgula para decimais)
   - Números grandes: 1.000 (ponto para milhares)
